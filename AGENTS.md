# AGENTS.md

## How to work in this repo
- Be conservative with changes.
- Prefer minimal diffs.
- Ask for approval before adding dependencies.
- Run tests relevant to changed files first.

## Code style
- Follow existing patterns in nearby files.
- Do not introduce new frameworks.
- Keep functions small and explicit.

## Safety checks
- Never modify CI, auth, or deployment files unless explicitly asked.
- Do not delete code unless the task requires it.

## Output expectations
- Explain the root cause briefly before proposing a fix.
- Summarize changed files and remaining risks.
